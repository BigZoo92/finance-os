# Agent Surfaces — Finance-OS

> Architecture of the multi-agent skill distribution system.

## Canonical Source (edit here only)

```
.agentic/source/skills/           100 skills, single source of truth
.agentic/source/references/       Heavy reference files (color-expert: ~985K tokens)
.agentic/manifests/               SHA-256 hash manifest
```

All skill editing happens in `.agentic/source/skills/`. No other location is authoritative.

## Generated Projections (never edit directly)

Every file in these directories is generated by `pnpm agent:skills:sync` and carries a header:

```markdown
<!-- GENERATED — DO NOT EDIT
     Source: .agentic/source/skills/adapt/SKILL.md
     Hash:   sha256:a1b2c3d4e5f6...
     Sync:   pnpm agent:skills:sync -->
```

| Target | Consumer | Directory Naming | Subset |
|---|---|---|---|
| `.claude/skills/` | Claude Code | Nested (`finance-os/core-invariants/`) | All skills + color-expert heavy references |
| `.agents/skills/` | Codex autopilot | Flat (`finance-os-core-invariants/`) | All except Claude-only meta skills |
| `.qwen/skills/` | Qwen | Flat | `make-interfaces-feel-better` only |
| `skills/` | Generic / Impeccable | Flat | 21 Impeccable UI subset |

## Commands

| Command | Purpose |
|---|---|
| `pnpm agent:skills:sync` | Project canonical to all 4 targets + write manifest |
| `pnpm agent:skills:check` | CI: verify all targets match manifest (exit 1 on drift) |
| `pnpm agent:skills:watch` | Dev: watch canonical, auto-resync on change |

## Workflow: Adding a New Skill

```bash
# 1. Create skill in canonical
mkdir -p .agentic/source/skills/my-new-skill
cat > .agentic/source/skills/my-new-skill/SKILL.md << 'EOF'
---
name: my-new-skill
description: What this skill does
---
# My New Skill
Instructions for the agent...
EOF

# 2. Sync to all targets
pnpm agent:skills:sync

# 3. Verify
pnpm agent:skills:check

# 4. Update skill routing if needed
# Edit scripts/agent-context/lib.mjs TASK_DOMAINS
```

## Workflow: Editing an Existing Skill

```bash
# 1. Edit ONLY in canonical
vim .agentic/source/skills/finance-os/core-invariants/SKILL.md

# 2. Sync
pnpm agent:skills:sync

# 3. Verify
pnpm agent:skills:check
```

## Heavy References

The `color-expert` skill has ~985K tokens of reference material (color theory, paint mixing, historical color standards). These live in `.agentic/source/references/color-expert/` and are injected only into the `.claude/skills/color-expert/references/` projection (Claude has large enough context). Other targets do not get heavy references.

## Cross-Platform Compatibility

- No symlinks — pure file copy
- SHA-256 hashes for integrity
- Works on macOS, Linux, Windows, GitHub Actions, cloud agents
- Manifest is JSON — parseable by any tool
