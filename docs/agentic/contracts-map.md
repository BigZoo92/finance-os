## Manual Contract Checks

- `GET /auth/me` should never 404 in prod or prod-like environments.
- `GET /health` and `GET /version` should stay schema-compatible across web, api, and the worker's localhost-only status server; `GET /healthz` remains a web compatibility alias.
- `GET /dashboard/goals` should return deterministic demo data in demo mode and must not require a DB read to do so.
- Goal writes should fail with `403`, never silently succeed, when the admin session is missing.
- `GET /dashboard/derived-recompute` should return deterministic snapshot metadata in demo mode and must not require a DB read to do so.
- `POST /dashboard/derived-recompute` should fail with `403`, `409`, or `503` safe envelopes when demo mode, overlap protection, or the feature flag blocks execution.
- `GET /dashboard/news` should return deterministic fixture data in demo mode and must not execute provider calls or DB writes; admin mode should keep the same contract shape and fall back to fixture-backed payloads on live/cache failures.
- `GET /dashboard/markets/overview` should return deterministic fixture data in demo mode and must not execute provider calls or DB writes; admin mode should keep the same contract shape and fall back to fixture-backed payloads on live/cache failures.
- `GET /dashboard/markets/watchlist`, `/macro`, and `/context-bundle` should stay cache-only and inherit the same freshness/provenance semantics as overview.
- `GET /dashboard/news/context` should stay cache-only and require admin or a valid internal token.
- `GET /dashboard/advisor` and all `GET /dashboard/advisor/*` artifact routes should stay deterministic in demo mode with no DB/provider calls and should never trigger live provider work on read.
- `POST /dashboard/advisor/manual-refresh-and-run` should fail with `403`, never silently run, in demo mode or without admin auth, and overlapping requests should return the active operation instead of starting a second mission.
- `POST /dashboard/advisor/chat` should stay deterministic and write-free in demo mode.
- `POST /dashboard/advisor/run-daily` and `POST /dashboard/advisor/relabel-transactions` should fail with `403`, never silently run, in demo mode or without admin/internal-token auth.
- `GET /dashboard/manual-assets` should return demo fixtures in demo mode and persisted assets only in admin mode.
- `POST/PATCH/DELETE /dashboard/manual-assets*` should fail with `403`, never silently write, in demo mode.
- `POST /dashboard/news/ingest` should return `403` in demo mode and `503` safe fallback envelopes on provider failure.
- `POST /dashboard/markets/refresh` should return `403` in demo mode and `503` safe fallback envelopes on provider failure.
- `POST /integrations/powens/callback` should fail with `401` or `403`, never a route miss, when auth/state is invalid.
- `POST /enrichment/bulk-triage` should return `503` with a safe fallback message when `ENRICHMENT_BULK_TRIAGE_ENABLED=false`.
- If a contract changes, update the caller, the nearest `AGENTS.md`, and this map in the same change.
